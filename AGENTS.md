# AGENTS.md

## 项目简介

本项目是一个在线判题系统（Online Judge）的后端判题沙箱服务。你是一名 C++ 开发助手，需要协助完成本服务的开发。

## 技术栈与环境

- **开发语言**：C++（标准不低于 C++17）
- **构建工具**：CMake
- **操作系统**：WSL2 (Ubuntu 26.04) 下开发
- **消息队列**：RabbitMQ，使用 `rabbitmq-c` 库（或 `SimpleAmqpClient`）
- **数据库**：MySQL，使用对应 C/C++ 连接库 `mysql-connector-c++`
- **沙箱**：基于 `isolate`（https://github.com/ioi/isolate）进行安全编译与运行
- **版本控制**：Git

## 服务职责

你是一个后台 Worker，不直接与前端交互。核心流程：

1. 从 RabbitMQ 队列中消费包含代码提交信息的消息。
2. 根据消息内容，调用沙箱编译并运行用户代码。
3. 将运行输出与标准测试用例进行比对，生成判题结果。
4. 将结果（状态、时间、内存、错误信息）写回数据库对应的提交记录中。
5. 确认消息（ACK）并继续下一条。

## 消息格式约定

假设 RabbitMQ 消息为 JSON 格式，包含以下字段：

```json
{
  "submission_id": 12345,
  "problem_id": 100,
  "language": "cpp",
  "code": "#include <iostream>...",
  "time_limit": 1000,
  "memory_limit": 262144,
  "testcases": [
    {"input": "1 2", "expected_output": "3"}
  ]
}
```

## 模糊信息约定

本项目为一个更大的项目的一部分。鉴于目前前后端交互界面尚未完全确定，所有涉及到前后端交互的信息格式你都可以自行决定，合理即可。包括但不限于：判题结果的具体格式、数据库表结构等。

项目完成后，需要生成一个readme.md文件，在其中详细描述项目的架构。同时，所有涉及到模糊信息的部分都需要在readme中专门使用一节来进行描述，方便后续确定接口后正式修改。

## 测试约定

项目完成后，请你搭建一个简单的测试框架，自拟RabbitMQ消息，测试判题沙箱的正确性。要求包含以下五种情况：

（1）运行成功。

（2）wrong answer。

（3）runtime error。

（4）运行超时。

（5）编译错误。

测试过程中使用到的具体算法程序可以自拟。
